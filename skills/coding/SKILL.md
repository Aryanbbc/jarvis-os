# Coding Skill

## Description
Use this skill when the user asks Jarvis OS to code, build, fix, debug, refactor, explain, review, or improve software projects.

## Trigger Words
- code
- build
- fix
- debug
- refactor
- error
- bug
- create
- implement
- setup
- improve
- optimize
- review

## Steps
1. Understand the user's coding request.
2. Identify the programming language, framework, and project context.
3. Inspect the error, requirement, or missing feature.
4. Plan the smallest working solution.
5. Write clean, complete, runnable code.
6. Avoid placeholders unless the user explicitly asks for them.
7. Explain file paths clearly when multiple files are needed.
8. Save important coding notes, fixes, and project decisions to the vault.
9. Update `vault/index.md` if a new note is created.

## Output Format

# Coding Task: <Task Name>

Date: <YYYY-MM-DD>

## Goal
<What needs to be built or fixed>

## Files Changed
- <file-path-1>
- <file-path-2>

## Solution
<Code or explanation of the implemented fix>

## Commands
<commands to run>

## Notes
<Important project notes>

## Next Actions
- <Recommended next step>

## Where To Save In Vault
Save coding notes to:

vault/reports/<YYYY-MM-DD>-coding-<task-slug>.md
